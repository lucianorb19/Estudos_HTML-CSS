Formulários

Formulários são meios de obter informações do usuário. Usa-se a tag form. De maneira simples, temos um formulário inicial


    form
    p Nome: input type="text" name="nome" id="inome"
    p Sobrenome input type="text" name="sobrenome" id="isobrenome"
    p input type="submit" value="Enviar"

Nome:

Sobrenome:

No formulário acima, não há uma relação entre o Nome e seu campo para preenchimento, assim como não há para Sobrenome. Isso se resolve usando a tag label para envelopar Nome / Sobrenome e no parâmetro for usar o valor que estiver em id da tag input


        form
        p label for="inome" Nome: 
        input type="text" name="nome" id="inome"
        p label for isobrenome Sobrenome 
        input type="text" name="sobrenome" id="isobrenome"
        p input type="submit" value="Enviar"

Métodos get e post

O parâmetro method da tag form assume dois valores, get e post, que são os métodos de envio da informação dos formulários para a página que vai utilizá-los. O get mostra os valores na url, o post não, porém, nenhum dos dois é seguro. Recomenda-se usar o get para infoções não sensíveis e o post para informações mais sigilosas. Além disso, o get é limitado a 3mil bytes, não deve ser usado para dados sensíveis e nem para envio de arquivos.

Alguns parâmetros úteis do form

  • autocomplete: define se vão ou não aparecer sugestões para os campos (preenchimentos anteriores)
  • action: o arquivo da linguagem que vai receber os dados. Por exemplo manda para cadastro.php
  • method: métodos de envio get e post
    • get: padrão - mostra os dados na url
    • post: não mostra os dados na url

Campos úteis do form

  • input: entrada do usuário
  • input type text: entra de texto do usuário
  • input type password: campo para senha
  • input type number: aceita números e -
  • step: para campos input number, define a progressão da parte decimal
  • input type month: um mês e ano (incompatível com mozilla, safari)
  • input type date: dia/mês/ano (incompatível com safari)
  • input type time: hora/minuto (incompatível com safari)
  • input type email: email (verifica se tem @)
  • input type tel: telefone celular
  • input type checkbox: campo para marcar/desmarcar. Aceita múltiplas marcações.
  • input type radio: campo para marcar/desmarcar. Aceita várias marcações. Caso seja preciso delimitar uma escolha somente, usar o mesmo valor nos inputs para o parâmetro name.
  • input type color: O usuário escolhe uma cor. Pode ser definida uma inicial no parâmetro value, assumindo valor headecimal.
  • input type range: escolha numa escala. Podem ser definidos min, max e valor inicial com value. A tag output auxilia a mostrar qual o valor que está sendo definido no range.
  • input type file: permite o envio de arquivos. Não sar com método get.
  • input type submit: botão para enviar as informações dos inputs
  • input type reset: botão que limpa os campos
  • name: útil para identificar o campo no HTML e PHP
  • id: útil para identificar o campo no JavaScript
  • required: não assume nenhuma valor. Se inserido no input, não vai enviar os dados caso o campo esteja vazio.
  • minlength: define o tamanho mínimo de caractéres para o campo
  • maxlength: define o tamanho máximo de caractéres para o campo
  • min: valor mínimo que o campo precisa assumir
  • max: valor máximo que o campo precisa assumir (usando o preenchimento da setinha. O usuário pode escrever um valor maior)
  • size: define o tamanho da caixa para mostrar x caractéres
  • placeholder: uma mensagem que vai aparecer dentro do campo antes do preenchimento
  • value: um valor inicial para o campo. Caso seja definido, sobrepõe o placeholder
  • autocomplete: dentro do input, define se o campo é um nome de usuário, senha nova, senha atual, dentre outras....
  • Tag fieldset: agrupa os campos dentro dela. Parâmetro legend dá o seu nome.
  • checked: deixa marcado o campo radio/checkbox
  • select: gera uma lista de opções para selecionar. Cada campo option value assume um valor. Podem ser agrupados com tag optgroup. Para definir um valor pré-selecionado, usar selected.
  • datalist: Cria uma lista de opções que pode ser associada pelo seu id ao campo list do input. Cada campo option assume um valor. Dessa forma, o usuário pode digitar a entrada ou escolher dentre as opções.
  • textarea: campos para mensagem. Prâmetro cols delimita quantos caracteres por linha e rows quantas linhas (mas na verdade é infinito).
  • output: mostra algo na tela, mas não envia pelo formulário. A ele podem ser aplicadas resultados de funções para mostrar algo específico. Uma soma, por exemplo. Ela pode ser usada como auxiliar para mostar o valor da tag range.
